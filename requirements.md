# 要件定義書：医療従事者向け勉強会・研修管理システム

対象：診療放射線技師・看護師・薬剤師・臨床検査技師など

| 項目 | 内容 |
|---|---|
| 文書名 | 医療従事者向け勉強会・研修管理システム 要件定義書 |
| 想定開発期間 | 2週間（MVP） |
| 想定技術 | Next.js + Supabase + Vercel |
| 初期検証対象 | 院外研修・学会・メーカー勉強会を管理する個人の医療従事者 |

## 1. プロジェクト概要

### 1.1 背景

医療従事者向けの勉強会・研修情報は、ラジくる、学会、職能団体、医療機器・製薬企業、地域の研究会など複数の媒体に分散している。そのため、興味のある研修を見つけても申込期限や開催日を忘れたり、同じ月に開催される研修を比較しにくかったり、受講後の記録や証明書が別々の場所に保存されたりする課題がある。

### 1.2 目的

- 研修カレンダーを中心に、研修情報、申込期限、参加予定、受講履歴、学習メモを一元管理できる状態を実現する。
- 医療従事者が継続的な学習を振り返り、次に学ぶ分野を判断しやすくする。
- MVPでは月間カレンダーと手動登録を中心とし、短期間かつ無料枠で利用可能なシステムを構築する。

### 1.3 スコープ

**含むもの**
- ユーザー登録・ログイン
- 月間研修カレンダー表示
- 研修情報の手動登録・編集・削除
- 日付・研修名クリックによる詳細表示
- 分野・開催形式による絞り込み
- 申込期限・申込状態の管理
- お気に入り・参加予定の管理
- 受講メモ・学びの記録
- 参加履歴・簡易集計

**含まないもの**
- 診断、治療方針、医薬品選択などの医療判断
- 資格単位の公式認定・自動判定
- 研修主催者サイトへの自動申込
- MVPでの外部サイト定期巡回・自動公開
- ログインが必要なJSRT会員ページへの自動ログイン
- 患者情報・診療情報の保存
- 施設全体の研修受講管理

## 2. ユーザー種別・ペルソナ

| ユーザー種別 | 主な利用場面 | 主な課題 | 優先度 |
|---|---|---|---|
| 診療放射線技師 | 核医学・CT・MRIなどの研修管理 | 情報分散、期限忘れ、参加履歴の把握 | 最優先 |
| 看護師 | 院内外研修・認定関連学習 | 勤務と日程調整、記録の分散 | 高 |
| 薬剤師 | 製薬企業・学会研修 | 単位・証明書・主催者情報の分散 | 中 |
| 臨床検査技師 | 専門領域別研修 | 地域・オンライン研修を探しにくい | 中 |
| 研修担当者 | 施設内研修管理 | 案内・申込・出欠の別管理 | 将来 |

## 3. ユーザーストーリー

- As a 医療従事者, I want 月間カレンダーで研修を確認したい, So that 開催日や空いている日を直感的に把握できる.
- As a 医療従事者, I want 研修情報を登録したい, So that 興味のある研修を見失わずに管理できる.
- As a 医療従事者, I want 申込期限を登録したい, So that 申込忘れを防げる.
- As a 医療従事者, I want 参加予定を開催日順に確認したい, So that 勤務や私生活との予定調整ができる.
- As a 医療従事者, I want 参加後に要点や学びを記録したい, So that 業務や復習に活用できる.
- As a 医療従事者, I want 過去の参加履歴を分野別に確認したい, So that 学習の偏りや実績を把握できる.
- As a 薬剤師, I want 受講証明書を研修に紐づけたい, So that 更新手続きの際に探しやすくなる.
- As a 研修担当者, I want 参加者や出欠を管理したい, So that 院内研修の運営を効率化できる.

## 5. 非機能要件

| 項目 | 要件 |
|---|---|
| パフォーマンス | 通常のページ読み込みを3秒以内とし、月間カレンダーの切り替えを2秒以内とする。 |
| セキュリティ | Email + パスワード認証。通信はHTTPS。Supabase RLSで本人データのみアクセス可能とする。 |
| プライバシー | 患者情報・診療情報を入力しない設計とし、研修・学習に必要な情報のみ保存する。 |
| 可用性 | 利用サービスの無料枠に準じ、目標稼働率99%とする。 |
| スケーラビリティ | 初期は同時接続100ユーザーを想定する。 |
| レスポンシブ | PCとスマートフォンの両方で主要操作を行える。 |
| アクセシビリティ | 文字サイズ、コントラスト、キーボード操作を考慮する。 |
| バックアップ | Supabaseの提供機能と定期的なCSV出力でデータ消失に備える。 |
| 情報鮮度 | 外部取得情報には情報元URL・取得日時・確認状態を表示し、公式案内を正とする。 |
| AI安全性 | AI生成内容は参考情報として表示し、公式案内の確認を促す。 |

## 6. 制約条件

- 予算：Vercel・Supabaseなどの無料枠を基本とする。
- 期間：MVPは2週間での実装を目標とする。
- 技術：Next.js、TypeScript、Supabase、Vercelを使用する。
- 情報取得：MVPでは研修情報を利用者が手動登録する。v1.0では公開URLから候補を生成する。
- 外部情報源：ラジくる（https://radicle.site/）とJSRT公開サイトを候補対象とする。
- 認証ページ：JSRT会員サービスなどログイン必須ページは自動取得対象外とし、認証情報を保存しない。
- 公開手順：外部取得情報は即時公開せず、管理者または利用者の確認後にカレンダーへ掲載する。
- 法務・安全：患者情報を扱わず、医療判断を行う機能を実装しない。
- 外部サイト：研修情報や申込条件は主催者の公式ページを正とする。

## 7. 画面遷移・サイトマップ

階層は3層以内とし、重要な機能へ2クリック以内でアクセスできる構造とする。

```
研修カレンダー（トップ）
├── カレンダー
│   ├── 月表示
│   ├── 週・一覧表示（v1.0）
│   └── 研修詳細
├── 研修を探す
│   ├── 条件検索
│   └── 外部研修候補（v1.0）
├── 参加予定
│   ├── お気に入り
│   ├── 申込済み
│   └── 申込期限一覧
├── 学習記録
│   ├── 参加履歴
│   ├── 受講メモ
│   └── 受講証明書（v1.0）
├── 研修登録
│   ├── 手動登録
│   └── URLから登録（v1.0）
├── マイページ
│   ├── アカウント設定
│   └── プロフィール
└── ヘルプ・お問い合わせ（v1.0）
```

### 7.1 主要画面遷移

- ログイン → 研修カレンダー → 日付または研修名を選択 → 研修詳細
- 研修カレンダー → 分野・開催形式を絞り込み → 研修詳細 → 申込ページ
- 研修詳細 → お気に入りまたは参加予定へ追加 → 申込済みに更新
- 研修カレンダー → 研修登録 → 手動入力 → カレンダーへ反映
- 学習記録 → 参加済み研修 → 受講メモ登録 → 参加履歴
- 外部研修候補（v1.0） → 内容確認・修正 → 承認 → カレンダーへ掲載

## 8. ナビゲーション設計

- サイドバー：研修カレンダー、研修を探す、参加予定、学習記録を固定表示する。
- ヘッダー：主要CTA「研修を登録」を常設し、カレンダー上から1クリックで登録画面へ移動できる。
- 右上：マイページとログアウトを配置する。
- 用語は「見つける→カレンダーで確認する→申し込む→参加する→記録する」という利用者の行動に合わせる。

## 9. 外部研修情報の取り込み方針（v1.0以降）

| 情報源 | 取得範囲 | 取り込み方法 | 注意事項 |
|---|---|---|---|
| ラジくる | ログイン不要で公開されている研修情報 | v1.0でURL取り込み・研修候補生成の対象 | 利用規約・robots.txt・アクセス負荷を確認する |
| JSRT公開サイト | 一般公開されているイベント・セミナー情報 | v1.0で研修候補生成の対象 | 公式詳細ページを正とする |
| JSRT会員サービス | ログイン後の会員向け情報 | 自動取得しない。利用者が手動登録する | ID・パスワードを保存しない |

### 9.1 取り込みフロー

1. 外部公開ページを確認し、研修情報を「未確認候補」として保存する。
2. 研修名、開催日時、申込期限、形式、会場、主催者、費用、申込URL、情報元URLを抽出する。
3. 管理者または利用者が内容を確認・修正する。
4. 承認後に研修カレンダーへ公開する。
5. 同一URLまたは類似する研修名・開催日時が存在する場合は重複警告を表示する。
6. 取得に失敗した場合は手動登録へ切り替える。

## 10. データ要件

| データ | 主な項目 | 備考 |
|---|---|---|
| ユーザー | ID、Email、職種、専門分野、地域 | 患者情報は保存しない |
| 研修 | 名称、概要、開催日、開始・終了時間、形式、場所、都道府県、主催者、対象職種、専門分野、参加費、定員、単位情報、申込URL、詳細URL | 公式ページURLを保存 |
| 情報源 | 情報元サイト、情報元URL、取得日時、確認状態、公開状態 | 外部取得情報の監査用 |
| 申込 | 申込期限、申込状態、申込日 | 未申込・申込済み・不要 |
| 参加予定 | 参加状態、準備メモ | 予定・参加済み・欠席 |
| 学習記録 | 要点、学び、業務への活用、資料URL | 自由記述 |
| 証明書 | ファイルURL、発行日 | v1.0 |

## 11. 受入基準（MVP）

- 新規ユーザーが登録・ログインできる。
- 利用者が研修情報を登録・編集・削除できる。
- 月間カレンダーに研修名が開催日ごとに表示される。
- 前月・次月・当月へ切り替えられる。
- 日付または研修名を選択すると詳細を確認できる。
- 分野と開催形式で表示を絞り込める。
- 申込期限と申込状態を登録できる。
- 参加予定を一覧で確認し、参加済み・欠席へ更新できる。
- 受講した研修に学習メモを保存できる。
- 参加履歴を年度・分野別に確認できる。
- 他の利用者のデータを閲覧・変更できない。
- PCとスマートフォンの両方で主要操作ができる。

## 12. 参考デザイン方針

- 医療・教育サービスとして、白・青・緑を基調に落ち着いたUIとする。
- トップページは月間研修カレンダーとし、日付内に研修名・開催形式・申込状態を簡潔に表示する。
- 研修詳細はモーダルまたは詳細ページで表示し、申込URL・お気に入り・参加予定の操作をまとめる。
- 研修の状態は、未申込・申込済み・参加予定・参加済み・欠席を色と文字で区別する。
- スマートフォンではサイドバーをドロワーメニューへ切り替える。
- AI生成結果には「公式情報を確認してください」と表示する。

## 13. 開発優先順位

- **MVP**：月間カレンダー、詳細表示、手動登録、絞り込み、申込状態、お気に入り、参加予定、学習メモ、履歴。
- **v1.0**：ラジくる・JSRT公開情報の候補取り込み、URL入力補助、通知、週・一覧表示、Googleカレンダー、証明書。
- **v2.0**：AI要約、おすすめ研修、中止・延期候補の検知、復習支援、学習改善提案。
